# 🎱 행운의 로또 (JavaScript Lotto)

바닐라 자바스크립트로 구현하는 로또 게임 프로젝트입니다. Step 1에서는 핵심 도메인 로직과 CLI 환경을 구축하였고, Step 2에서는 기존 도메인을 보존하면서 웹 UI를 결합하는 과정에 집중했습니다.

## 🚀 데모 페이지

[행운의 로또 바로가기](https://goodsmell.github.io/javascript-lotto)

## 🛠 주요 기능

**1. 로또 구입**

- 사용자는 구입 금액을 입력할 수 있습니다 (단위: 1,000원).

- 최대 구입 금액 제한: 과도한 구매 방지를 위해 최대 100,000원까지만 입력 가능합니다.

- 구입 금액에 해당하는 만큼의 로또가 자동 발급됩니다.

**2. 로또 관리**

- 발급된 로또의 개수와 각 로또의 번호를 화면에 출력합니다.

- 도메인 로직을 통해 중복 없는 1~45 사이의 숫자 6개가 정렬되어 생성됩니다.

**3. 당첨 결과 확인**

- 당첨 번호(6개)와 보너스 번호(1개)를 입력합니다.

- 당첨 번호에 대한 유효성 검사를 수행합니다 (중복, 범위, 개수 등).

- 결과 확인 버튼을 누르면 당첨 통계와 총 수익률이 포함된 모달이 나타납니다.

**4. 당첨 통계 및 수익률**

- 1등부터 5등까지의 당첨 개수를 계산합니다.

- 수익률 계산: 총 수익률을 계산하며, 소수점 둘째 자리에서 반올림하여 첫째 자리까지 표시합니다.

- 금액 포맷팅: 당첨 금액은 한국 통화 기준에 맞춰 3자리마다 쉼표(,)를 표시합니다.

## 🏗 프로젝트 구조

```
src/
├── model/ # 비즈니스 로직 (Money, Lotto, LottoStore, LottoResult 등)
├── service/ # 도메인 모델들을 조합하는 서비스 레이어 (LottoMachine)
├── view/ # UI 렌더링 및 DOM 이벤트 처리
│ ├── console/ # CLI용 View (Step 1)
│ └── web/ # Web용 View (Step 2 - MoneyInput, LottoList, ResultModal 등)
├── styles/ # BEM 방법론을 적용한 모듈형 CSS
└── utils.js # 공통 유틸리티 (포맷팅, 반올림 등)
```

## 💻 실행 방법

**설치**

```
npm install
```

**웹 실행 (Vite)**

```
npm run start-step2
```

**테스트 실행 (Jest)**

```
npm run test
```

**배포**

```
npm run deploy
```

## ✅ 프로그래밍 요구 사항 및 학습 목표

- 도메인과 UI 분리: 도메인 영역의 코드를 UI 관련 코드로부터 독립시켜 설계의 순수성을 유지.

- TDD (Test Driven Development): 핵심 도메인 로직에 대해 Jest를 활용한 단위 테스트 수행.

- BEM 네이밍: CSS 클래스 선택자 명명 시 BEM 규칙을 준수하여 유지보수성 확보.

- 시맨틱 마크업: 웹 표준을 준수하는 태그 사용으로 구조적 완성도 향상.

## 💬 논의 및 회고

- 컨트롤러 분리: 입출력 환경이 다른 콘솔(step1-app)과 웹(step2-app)의 컨트롤러를 분리하여 각 환경에 최적화된 로직을 구성했습니다.

- UI TDD: 현재 도메인 위주의 테스트가 진행되었으나, 향후 DOM 렌더링 및 이벤트에 대한 테스트 자동화 방안을 모색 중입니다.
